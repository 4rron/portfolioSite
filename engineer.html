Grass Material

UE4 Material/Shader Development

The main purpose of this project was to create the most efficient grass possible. This fascination sparked inside of me the first time I played The Legend of Zelda: Breath of the Wild. The grass in this game was so full yet so well optimized that the player could react with it and the Nintendo Switch, even though limited in processing power, would not stutter. There was a progression in how I created the grass (from individual meshes for each blade of grass, to a two-plane setup, to a single-plane setup). This matched with coding interactions in the material blueprint itself instead of creating physics-based interactions helps optimize the grass to be as efficient yet beautiful as possible.

Description

This grass was a culmination of studies into how to build grass foliage based off a two-plane mesh and hand-drawn texture. All interactions with the player are done through the object's shader itself, not through game programming.

Tools Used

  • Unreal Engine 4
  • Adobe Photoshop

Skills to Highlight

This grass shows off my in depth understanding of shader programming and development along with my passion of stylized graphics.

Refinement Plan

Proposed Changes

The main change I wish to make to this material is to figure out how to properly control the shader so that the grass actually moves entirely away from the player instead of stabbing into the player's legs. There are a couple other optimizations that need to be done to make the material a bit easier to light, but actual interactivity is my main focus, at least during this study of it.

Skills Showcase

  • Unreal Engine 4 Blueprints
  • Material Blueprints
  • Adobe Photoshop
  • Shaders

Dart

3D Modeling and Animation

This artifact began originally as a study into animation in Autodesk Maya. With that being said, I have slowly attempted to convert over to a completely free workflow, so I have modeled my new projectile in Blender. This new projectile is based off of a foam dart (commonly found in © Nerf toys).

I have created realistic physics on these models within Unreal Engine 4, including a randomized stick function on the window for the darts.

I also created a system stress test out of the darts to see how many a computer could simulate. This was more for my own interest after taking the time to create the dart and the physics, but I thought it was interesting, nonetheless.

I investigated skeletal setups in 3D modeling software along with Unreal's animation features. It is still not complete, but I am trying to programmatically animate the darts based off of impact angle.
